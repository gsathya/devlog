Devlog

Updated the devlog. There was a lot of problems with my previous devlog approach.

  • The whole rsync-ing from the librelist archives dance was painful
  • I couldn't update the post once I sent the mail.
  • Too many hits, and I had to pay $$$ for my ec2.
  • Couldn't see the output before sending it.
  • Posts weren't sorted according to date of publishing.

Well, now I've updated the devlog to take care of these things. My workflow is now -

  • Write post in markdown and save it locally in /drafts
  • Send the content to the list
  • Run python devlog.py --publish, commit new posts in /site/posts, push code to github.
  • ???
  • Profit.

It's a lot more steps now, but it gives me more flexibility than my previous approach. Github now hosts the site, so I don't pay for the hosting. Posts are now sorted by date. And for bonus, it supports markdown. Screw you textile.
